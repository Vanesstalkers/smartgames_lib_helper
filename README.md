# lib/helper

Вспомогательный модуль: пошаговый туториал (helper), диалоги и API для восстановления ссылок.

## Структура

### Логика туториала

- **getTutorial.js** — получение объекта туториала по пути (domain/lib → steps). Используется для отображения подсказок по шагам.
- **updateTutorial.js** — обновление состояния туториала (текущий шаг, завершение).

### API (`api/`)

- **restoreLinks.js** — восстановление ссылок (например, после перезагрузки страницы).
- **action.js** — API-действия, связанные с хелпером (вызов с клиента).

### Фронт (`front/`)

- **helper.vue** — UI хелпера (подсказки, шаги туториала).
- **components/dialog.vue** — переиспользуемый диалог.

### Документация (`docs/`)

- **alert-list.md** — список алертов/уведомлений.
- **helper-link.md** — описание ссылок и форматов для хелпера.

## Связи

- Туториалы в `lib/lobby/tutorial` и `lib/game/tutorial` задают шаги; хелпер отображает их через `getTutorial` и обновляет состояние через `updateTutorial`.
- Доменные туториалы (например, `domain/game/tutorial`) могут добавлять свои шаги и ссылки.
